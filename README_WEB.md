# 🎓 ВКР Валидатор — Веб-сервис проверки оформления выпускных работ

## 🌟 Описание

Интуитивно понятный веб-сервис для автоматической проверки оформления выпускных квалификационных работ (ВКР) по требованиям методички ИГУ.

### ✨ Преимущества нового интерфейса:

1. **Понятная загрузка файла** — перетащите файл мышкой или кликните для выбора
2. **Визуальная обратная связь** — видите название и размер загруженного файла
3. **Индикатор загрузки** — показывает процесс проверки документа
4. **Результаты на русском языке** — все ошибки описаны понятным языком
5. **Чёткая локализация** — указано точное место ошибки в документе (глава, параграф)
6. **Рекомендации по исправлению** — для каждой ошибки дана инструкция как исправить
7. **Цветовая дифференциация** — ошибки, предупреждения и информация различаются по цвету

## 📋 Как пользоваться

### Шаг 1: Загрузка файла
- Перетащите файл `.docx` в область загрузки
- Или кликните по области и выберите файл через диалог

### Шаг 2: Проверка файла
- После загрузки файла нажмите кнопку **"🔍 Проверить документ"**
- Дождитесь завершения проверки (может занять несколько минут)

### Шаг 3: Просмотр результатов
Вы увидите:
- **Сводную статистику** — общее количество ошибок по категориям
- **Список ошибок** — если они найдены
- **Рекомендации** — как исправить каждую ошибку

---

## 📊 Структура отчёта об ошибках

Каждая ошибка содержит:

| Поле | Описание |
|------|----------|
| **Код ошибки** | Например: "Ф-1", "С-5", "Л-7" |
| **Тип** | Ошибка / Предупреждение / Информация |
| **Место** | Где найдено: "Глава 1 > Параграф 1.2" |
| **Правило** | Формулировка требования из методички |
| **Найдено** | Что фактически обнаружено в документе |
| **Ожидалось** | Что должно быть по требованиям |
| **Фрагмент** | Цитата из документа (до 100 символов) |
| **Рекомендация** | Инструкция по исправлению |
| **Источник** | Ссылка на раздел методички |

## 🔧 API Endpoints

Если вы хотите использовать сервис программно:

### `GET /`
Главная страница с веб-интерфейсом

### `POST /api/validate`
Загрузить файл для валидации

**Пример запроса:**
```bash
curl -X POST http://localhost:8000/api/validate \
  -F "file=@work.docx"
```

**Пример ответа:**
```json
{
  "doc_id": "uuid...",
  "summary": {
    "total_errors": 5,
    "formatting": 3,
    "style": 1,
    "citations": 1
  },
  "errors": [
    {
      "code": "Ф-1",
      "type": "formatting",
      "severity": "error",
      "location": {
        "paragraph_index": 15,
        "structural_path": "Глава 1 > Параграф 1.2"
      },
      "rule": "Шрифт основного текста должен быть Times New Roman, 14 пт",
      "found_value": "Arial, 12 пт",
      "expected_value": "Times New Roman, 14 пт",
      "recommendation": "Выделите весь текст и измените шрифт на Times New Roman, размер 14 пт",
      "rule_citation": "§4.2, с. 47"
    }
  ]
}
```

### `GET /health`
Проверка работоспособности сервиса

### `GET /docs`
Swagger UI документация API

## ✅ Проверяемые требования

Сервис проверяет документ по следующим группам требований:

### Форматирование (Ф-)
- Шрифт (Times New Roman, 14 пт)
- Межстрочный интервал (1,5)
- Выравнивание (по ширине)
- Поля (левое 3 см, правое 1 см, верхнее/нижнее 2 см)
- Абзацный отступ (1,25 см)

### Структура (С-)
- Наличие обязательных разделов
- Нумерация глав и параграфов
- Оформление заголовков
- Разрывы страниц

### Таблицы и рисунки (Т-)
- Оформление подписей
- Нумерация
- Шрифт внутри таблиц
- Выравнивание

### Список литературы (Л-)
- Формат ссылок [N]
- Алфавитный порядок
- Оформление авторов
- Дата обращения для URL

## 📝 Примеры использования

### Через веб-интерфейс:
1. Откройте http://localhost:8000
2. Перетащите файл work.docx
3. Нажмите "Проверить документ"
4. Изучите отчёт

### Через командную строку:
```bash
# Загрузить файл и получить JSON-отчёт
curl -X POST http://localhost:8000/api/validate \
  -F "file=@diplom.docx" > report.json

# Проверить статус сервера
curl http://localhost:8000/health
```

### Через Python:
```python
import requests

with open('work.docx', 'rb') as f:
    response = requests.post(
        'http://localhost:8000/api/validate',
        files={'file': f}
    )
    
report = response.json()
print(f"Найдено ошибок: {report['summary']['total_errors']}")

for error in report['errors']:
    print(f"\n{error['code']}: {error['rule']}")
    print(f"Место: {error['location']['structural_path']}")
    print(f"Как исправить: {error['recommendation']}")
```

## ❓ Часто задаваемые вопросы

### Q: Какие форматы файлов поддерживаются?
**A:** Только `.docx` (Microsoft Word 2007+)

### Q: Как долго длится проверка?
**A:** Обычно 10-60 секунд в зависимости от размера документа

### Q: Можно ли проверить работу через консоль?
**A:** Да, используйте `curl` или Python (см. примеры выше)

### Q: Что делать, если найдены ошибки?
**A:** Для каждой ошибки дана рекомендация по исправлению. Следуйте инструкциям в отчёте.

### Q: Точно ли определены все ошибки?
**A:** Сервис проверяет формальные требования. Некоторые нюансы могут потребовать ручной проверки.
